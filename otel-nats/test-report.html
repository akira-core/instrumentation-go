env_flags_test.go 2 Truth tables for both gates: (1) two-tier tracing (OTEL_INSTRUMENTATION_GO_TRACING_ENABLED × OTEL_NATS_TRACING_ENABLED) and (2) three-tier propagation (× OTEL_NATS_PROPAGATION_ENABLED). Covers truthy/falsy synonyms (true/1/on/yes/""/arbitrary; false/0/no/off/padded). Per-row t.Setenv + os.Unsetenv; calls resetNATSGateForTest() after each row to re-arm sync.Once cache. Every per-call decision in the traced impl ultimately consults one of these two gates. If the truth table regresses, the entire disabled-mode invariant fails silently.
propagation_test.go 2 HeaderCarrier.Get/Set/Keys against both nil nats.Header and a populated one. Direct struct construction; no NATS server needed. HeaderCarrier is the adapter between OTel's TextMapPropagator and NATS message headers — a regression here breaks every Publish header inject and every Subscribe extract.
conn_test.go 27 The bulk of the package: W3C round-trip, Publish/PublishMsg/Request producer spans, Subscribe/QueueSubscribe consumer spans, deliver span emission with the three-tier sampling gate (upstream sampled / local sampled / endpoint set), tracing-off skips both spans and headers, propagation-off keeps spans but skips headers, deliver-TP service name resolution, reply-link respects sampler. Real NATS server via natssrv.NewServer on 127.0.0.1:-1 (random port). In-memory tracetest.SpanRecorder. Helper waitSpanByNameAndKind polls with require.Eventually so async End() races are tolerated. NATS is wire-protocol over TCP — only an end-to-end server test can prove (a) headers actually appear on wire bytes and (b) subscribe handlers actually fire with the extracted context. This is the load-bearing file for the whole module.
flag_matrix_test.go 1 (8 sub-rows) All 8 combinations of (global × tracing × propagation) end-to-end against a real NATS server. For each: assert Conn.TracingEnabled() + PropagationEnabled() resolve to expected gate values; verify subscribe handler receives traceparent header iff propagation on; verify caller TP has at least one wrapper span iff tracing on (zero otherwise). Fresh natssrv instance per row, table-driven; otelnats.ResetGatesForTest() after each row. Span quiet-window assertion uses assert.Never(300ms, 10ms tick) instead of fixed sleep — fast-fail on first observed span. This file is the single bridge that ties the env truth tables to wire-observable behaviour. Without it, the gate functions could be correct but Publish could still inject headers (the v0.4.x regression that tracedJSImpl.PublishMsg ignored the propagation gate was exactly this class of bug).
connect_test.go 7 (1) Connect returns error on unreachable URL; (2) nil nats.Options are filtered; (3) Close is noop without deliver-TP; (4) Close honours bounded 3-second deliver-TP shutdown cap; (5) Drain doesn't block on slow exporter; (6) WithTraceDestination survives in tracing-on mode and (7) is dropped in disabled mode. Real natssrv + closed port (127.0.0.1:1) as OTLP endpoint to simulate slow exporter without setting up a mock OTLP receiver. Connection lifecycle bugs are slow to surface — a deliver-TP that fails to shutdown leaks an exporter batch goroutine per *Conn. Bounded-time assertions catch this within 5 seconds during CI rather than at production OOM.
traceevent_test.go 4 SubscribeTraceEvents (NATS 2.11+ infrastructure events): (1) emits 2 hop spans (ingress + egress) for a synthetic TraceEvent payload; (2) disabled-mode emits zero spans regardless; (3) malformed JSON payload does NOT crash or emit; (4) WithTraceDestination option round-trips through conn.TraceDest(). Synthetic TraceEvent JSON payload published directly via conn.NatsConn().Publish (bypassing the wrapper for the test fixture). Quiet-window negative assertions use assert.Never(300ms). Infrastructure trace events are an opt-in feature requiring NATS 2.11+; without these tests the only way to exercise them is full e2e against a real cluster. Direct payload injection is the only way to unit-test the hop-emission logic.
