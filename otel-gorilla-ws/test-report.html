Unit Tests (root package)

env_flags_test.go 1 2-tier flag gate truth table: OTEL_INSTRUMENTATION_GO_TRACING_ENABLED × OTEL_GORILLA_WS_TRACING_ENABLED. ~20 rows covering truthy/falsy synonyms. No propagation flag (intentional — envelope is inline; subprotocol negotiation serves that role). Direct calls to wsTracingEnabled(); per-row t.Setenv + os.Unsetenv; resetWSGateForTest() after each row. This is the load-bearing gate for the per-connection impl selection in newConn. A regression flips every newly-created Conn into the wrong (or right but should-not-be-tracing) path.
message_test.go 9 Cross-language wire format contract: MarshalWire produces the exact {"header":{...},"data":...} shape JS clients expect; TryUnmarshalWire accepts both the envelope AND the legacy flat format for backward compat; non-JSON payloads are wrapped as JSON-encoded strings; sync.Pool buffer reuse safety across goroutines. Hand-built byte arrays compared with assert.JSONEq; goroutine fan-out for pool reuse stress. This file is the SINGLE GUARDIAN of the cross-language wire contract. Any change to the envelope shape breaks JS clients silently — the only way to catch this without a JS test rig is exact byte-level assertions in Go.
options_test.go 5 resolveOptions falls back to global Tracer/Propagator when no options passed; custom options don't mutate globals; NewConn selects internal/direct.Conn when feature disabled, internal/traced.Conn when enabled; negotiated=false (passthrough subprotocol) keeps tracing on but disables PropagationEnabled. Recorder-based: assert which TP a custom-option tracer.Start writes to vs the global TP. Functional options are the ONLY way callers customise the wrapper without monkeypatching globals. If resolveOptions wrongly mutated globals, instrumenting multiple connections with different propagators would have ordering-dependent behaviour.
helpers_test.go 4 Error type predicates (close error detection, normal vs abnormal closures), formatting helpers. Direct calls to helpers in helpers.go. These helpers run in error paths — bugs are invisible until a real disconnect happens in production. Direct unit coverage catches off-by-one in close-code classification.
conn_test.go 9 (1) TestRoundTrip + TestSpanAttributes — end-to-end envelope round-trip with span attributes verification; (2) Scenarios C, D, E (client-side dial scenarios — see tests/integration/handshake_test.go for the full A–H table); (3) Scenarios F, G, G_FromPrefixedClientToken, H (server-side upgrader scenarios). httptest.NewServer with both otelgorillaws.Upgrader and plain websocket.Upgrader handler variants; tracetest.SpanRecorder for span observation. Subprotocol negotiation is THE central correctness concern of this wrapper — there are 8 distinct scenarios (A–H per README). Each is a different decision in the upgrade path. Without per-scenario tests, regressions in one branch don't surface until a specific client mix hits it in production.
upgrader_test.go 5 (30 sub-cases) Pure-function unit tests for the upgrader's internal helpers: splitClientProtocols (9 cases including bare otel-ws, prefixed, mixed), selectFirst (7 cases including accept-any semantics), cloneHeader (3 cases including nil-safety + independence), appProtocolFromRaw (5 cases for prefix stripping), isOTelWireProtocol (6 cases including case-sensitivity). Internal-package tests (no _test suffix on package) — these are pure functions, no I/O, no goroutines. The high-level Scenario tests in conn_test.go indirectly cover these but a 9-case table per pure function is cheaper to write, faster to run, and pinpoints the bug to a specific function rather than a whole handshake path.
flag_matrix_test.go 4 (1) TestWSFlagMatrix_AllCombinations — 6 env combinations × end-to-end client/server with envelope-or-not assertion; (2) TestWriteMessageDisabled_ByteIdenticalWithNative — runs the SAME payload through both otelgorillaws.Dial (disabled mode) and raw websocket.Dialer, asserts the server receives byte-identical bytes from both; (3) TestWriteMessageEnabled_NonNegotiatedStillByteIdentical — envelope MUST be suppressed when subprotocol negotiation fails even if env is on; (4) TestSubprotocolStripsOTelPrefix — Conn.Subprotocol() strips otel-ws+ prefix. Internal-package test so we can reset the unexported wsGate directly. httptest server with both gorilla and otelgorillaws upgraders. Cleanup ordered carefully (gate reset last via LIFO) so hijacked WebSocket handler goroutines don't race with the gate reset. The byte-identical assertion is the strongest possible disabled-mode invariant proof — it runs both paths and compares wire bytes. If anything in the disabled path ever started wrapping payloads (even accidentally), this test fails immediately.

error_fallback_test.go 1 Non-envelope payloads + close messages don't crash the wrapper — TryUnmarshalWire gracefully returns false and ReadMessage passes the raw bytes through. Server sends a non-JSON payload then closes; assert client doesn't error or panic. Real-world WebSocket traffic includes ping/pong/close frames and clients may send unwrapped payloads — robustness here is the difference between a crashy library and a useful one.
