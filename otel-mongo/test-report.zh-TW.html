測試檔案 — 測什麼 / 怎麼測 / 為何測

下表列出 v1 與 v2 中每個 *_test.go 及其理由。Parity 欄表示另一版本是否有對應檔。

檔案（v1 + v2 都有，除非註明） 測試數 Parity 測什麼 怎麼測 為何測
env_flags_test.go 4 v1 ≡ v2 三層 flag gate 真值表：OTEL_INSTRUMENTATION_GO_TRACING_ENABLED × OTEL_MONGO_TRACING_ENABLED × OTEL_MONGO_PROPAGATION_ENABLED。約 30 列涵蓋 truthy / falsy 同義詞（true / 1 / on / yes / "" / 任意字串；false / 0 / no / off / 含空白）。 直接呼叫 mongoTracingEnabled()、mongoPropagationEnabled()、resolveFlag()、resolveDocumentPropagation()，每列用 t.Setenv + os.Unsetenv。 「flag off → 零 spans、零 wire 影響」的 invariant 從這三個 gate 散開。若任一列回歸，下游 wrapper 程式碼就會誤入 SDK 路徑。
env_flags_testhelper_test.go 0 v1 ≡ v2 定義 resetPropEnabledCacheForTest() — 重新啟動 sync.Once-cached propagation gate 的測試專用 helper。 _test.go 內的普通函式（套件範圍，未匯出）。 Cached gate 會把 env 在首次讀取時凍結，若無此 helper，後續 t.Setenv 都失效。
flag_matrix_test.go 2 v1 ≡ v2 8 個 (global × tracing × propagation) 組合的端到端 matrix：(1) NewCollection 回傳正確 impl 型別 — 任何 gate off → *direct.Collection，全 on → *traced.Collection 且 PropagationEnabled bit 正確；(2) ContextFromDocument 在任何 gate off 時拒絕有效的 _oteltrace 內容。 Table-driven 8 列 × strategy-impl 型別斷言 + 透過 cachedPropagationEnabled() 驗證 cache。 單獨的 gate 函式測試無法證明「WRAPPER 真的有跟著 gate 走」。這個檔是從真值表通往可觀察 wrapper 行為的橋梁 — 唯一鎖住「flag off ⇒ 無 SDK 可達」的測試。
tracing_test.go 6 / 7 v1 ≡ v2 公開 API ContextFromDocument / ContextFromRawDocument 對所有 BSON 形狀（bson.M / map[string]any / bson.D / struct 慢路徑）；跨型別 malformed traceparent 拒絕；disabled traced.Collection 的 StartDeliverSpan。含 $setOnInsert 內 dot-notation key 保留回歸測試（"u._id" 必須能完整通過 marshal/unmarshal round-trip）。 用 bson.Marshal 建構 raw document，呼叫公開 extractor，搭配 enableDocumentPropagation helper + resetPropEnabledCacheForTest cleanup 切換 flag 組合。 這是 _oteltrace propagation 的讀取側。所有 trace context 消費者（change-stream reader、MongoDB → NATS bridge、下游服務）都走這些 helper。回歸會悄悄破壞端到端 trace 連續性。
client_test.go 4 / 1 v1 較廣 v1：initMongoProvider 有/無 OTEL_EXPORTER_OTLP_ENDPOINT、mongoServiceName 格式、parseServerFromClientOptions 對 nil / 無 URI / 有 URI 輸入。v2：精簡版（URI 解析發生在 options.Client() 層，較單純）。 直接呼叫套件私有 helper；不需真實 mongod。 Connect 時的 semconv 屬性（server.address、server.port）與 deliver-TP gate 決策都在這裡做。若 mongoServiceName 格式化錯誤，整個 service graph 顯示錯誤 broker 標籤。
client_deliver_test.go 5 / 10 v1 ≡ v2（佈局不同） (1) traced.ClientState.ShutdownDeliver 對 nil MongoTP（noop）、真 provider（呼叫 processor Shutdown）、processor 阻塞時的 3 秒 timeout cap；(2) TestClientDisabledPathHasNoDeliverState 斷言 env=0 時 mongoTracingEnabled() 解析為 false；(3) v2 額外含 initMongoProvider + StartDeliverSpan 測試（合在同檔）。 客製化 shutdownObservingProcessor 帶可選 block 時間，滿足 SpanProcessor 介面；直接呼叫 state.ShutdownDeliver 斷言。 若 Disconnect 沒乾淨關掉 deliver-TP，每個建立短命 *Client 的長期服務都會每連線洩漏一個 exporter batch goroutine — 在 production OOM 前都看不到。
cursor_test.go 7 / 10 v1 ≡ v2 Cursor.Decode direct/traced 路徑；SingleResult.Decode / Raw / TraceContext 的 first-touch-ends-span 契約 6 個交叉排列 sub-cases（decode×2、decode→raw、raw→decode、decode→tracectx、tracectx→raw、三者全來）；passthrough impl 不發 span。 透過 mongo.NewCursorFromDocuments / mongo.NewSingleResultFromDocument 建構 cursor — 不需真實 mongod。用 injectTraceIntoDocument helper 建有 _oteltrace 的 document。 「Decode/Raw/TraceContext 首觸即終結 span」的契約很容易被悄悄破壞 — 未來重構加入早期 return 路徑可能讓 span 結束兩次（panic）或零次（洩漏）。交叉排列 cases 把這個釘死。
cursor_decodewithcontext_test.go（僅 v1） 2 僅 v1 v1 專屬行為：DecodeWithContext 創一個帶新 TraceID 的 INTERNAL span 並 link 到原始 trace（legacy 語義）。也斷言 disabled 路徑不發 span。 Recorder-based：建有 _oteltrace 的 cursor → decode → 檢視 tracetest.SpanRecorder.Ended() 的 mongo.cursor.decode span 與其 link。 v2 把 DecodeWithContext 改為純 context 增強（不再 emit 額外 span）。這個檔是唯一守住 v1 legacy 契約的測試；若無此檔，v1 會悄悄回歸為 v2 行為而無任何其他測試報錯。
collection_test.go 14 v1 ≡ v2 整合套件（testcontainers MongoDB）：InsertOne/InsertMany/ReplaceOne/UpdateOne/DeleteOne/CountDocuments/UpdateByID/DeleteOneByID/FindOneByID/FindByIDs/BulkWrite，全都斷言寫入時 _oteltrace 存在 + global-off 零 spans 路徑。含 TestNewCollection 驗證 strategy-split 決策。 tcmongo.Run(ctx, "mongo:7.0") 在 TestMain 啟動單機 MongoDB container，所有測試共用。requireMongoDB fallback 到 MONGO_URI 環境變數，Docker 不可用時 skip。 若無整合測試，wrapper 沒有端到端信心 — driver 行為太微妙（projection 規則、原子 op、write concern 等）無法忠實 mock。真實讀取時 _oteltrace 欄位的存在是 wrapper 確實穿越 MongoDB BSON 序列化的唯一有意義證據。
database_test.go 3 v1 ≡ v2 Gate 繼承鏈：ClientState.ForDatabase() → DatabaseState.NewCollection() 完整傳遞 Tracer / Propagator / PropagationEnabled / DeliverTracer / ServerAddr / ServerPort。也驗證 PropagationEnabled=false 沿鏈傳遞。 直接 struct 建構；不需真實 mongod。斷言 interface 值相等。 若繼承鏈漏一個欄位，透過 Client 建出的 Database/Collection 中途可能丟掉 propagation 設定，造成 Client/Collection 直接用 vs 經 Database.Collection 行為不一致。
internal/shared/extract_metadata_test.go 7 + 1 bench v1 ≡ v2 ExtractMetadataFromRaw 直接 lookup 與舊反射路徑 parity、缺欄位、錯誤型別 subdocument、空 traceparent；bson.D 反向掃描：位置不變（head/middle/tail）、reverse-scan 取最後一個（鎖住優化）、tail vs head benchmark。 用 bson.Marshal 建 bson.D 與 bson.Raw；直接呼叫底層 extractor。 這是最熱讀取路徑（change-stream 迭代）。若 reverse 改成 forward scan 會悄悄拖慢讀取，並破壞「wrapper 注入最後一個 entry 勝出」契約 — 這是 injectTraceIntoDocument 的 inject/extract 對稱性所依賴的。
internal/shared/inject_trace_test.go 14+ v1 ≡ v2 v0.5.2 sampling gate：14 個 case 證明 unsampled 寫入完全跳過 _oteltrace。加上呼叫者 document 不被 mutate（immutability）、所有四種輸入型別（bson.D / bson.M / map / struct）。 用明確 sampler 選擇建 context；呼叫 InjectTraceIntoDocument；斷言結果中 _oteltrace 的存在/缺席。 v0.5.2 新增 sampling gate 是破壞性預設變更。這個檔是負責證明新契約在所有輸入形狀都成立的關鍵測試。
internal/shared/inspect_writemodel_test.go 多個 v1 ≡ v2 BulkWrite 操作：InsertOneModel、UpdateOneModel、$setOnInsert、model 不可變性。 對 driver bulk-write model 型別的 reflection 風格 helper。 BulkWrite 是單一 round-trip 包多 op — model 中途丟掉 _oteltrace 會悄悄讓一半操作失去 trace 連續性。
internal/shared/bulkwrite_test.go 多個 v1 ≡ v2 BulkWrite 端到端 inject 行為；與 model 檢視器配合。 直接呼叫 internal helper；不需真實 mongod。 Bulk 操作是最容易因 per-model 處理而丟掉 trace context 的路徑 — 明確覆蓋避免靜默丟失。
internal/shared/upsertset_test.go（僅 v2） 1 僅 v2 v2 專屬 upsertSetField helper 處理 mongo-driver 2.x 的 SetOptions pattern。 直接 unit test。 v2 driver 引入新的 option-builder pattern；無此測試則 v1 與 v2 upsert 路徑行為漂移無法被偵測。
internal/traced/changestream_test.go 2 v1 ≡ v2 buildConsumerCtx：建立新 TraceID 並透過 span-link 連到原始 trace；當 DeliverTracer 設置時，deliver-tracer 成為 change-stream span 的父節點。 直接用 tracer + deliver tracer 建構 traced.ChangeStream；透過 recorder 斷言 span links 與 parentage。 Change stream 是最微妙的 propagation case — 生產者（writer）與消費者（watcher）在時間與行程上都解耦。Span links 而非 parent-child 保留 causality 而不暗示同步巢狀。
internal/flags/flags_test.go 5 v1 ≡ v2（byte-identical） EnvEnabled truthy/falsy 同義詞；Gate.Enabled 透過 sync.Once 確保 resolver 只跑一次；ResetForTest 重新啟動；併發 Enabled 讀取安全。 直接 unit test 打 flags 原語 — 不在 t.Setenv 範圍外動 env。 此檔在四個 sibling module 之間 BYTE-IDENTICAL（CI drift-check job 強制）。守護每個 wrapper 依賴的核心 flag 原語；這裡回歸就是四模組同時回歸。
internal_aliases_test.go 0 v1 ≡ v2 把 internal 型別（injectTraceIntoDocument 等）重匯出到 facade _test.go namespace，讓測試可以呼叫而不打破 internal/ 套件邊界。 型別 alias + 函式 variable 重綁定。 無此檔則 facade 套件的測試無法直接驅動 internal helper — 每個測試都要透過公開 API 建構，無法快速 unit test。
